# Oracle Cloud Free Tier 배포 가이드

## STEP 1. VM 생성

Oracle Cloud 콘솔 → Compute → Instances → Create instance

| 항목 | 설정값 |
|------|--------|
| Name | goldsangsa |
| Image | Canonical Ubuntu 22.04 |
| Shape | **Ampere → VM.Standard.A1.Flex** (Always Free) |
| OCPU | 2 |
| Memory | 12GB |
| SSH Key | **새로 생성 후 .pem 파일 다운로드** (분실 시 재접속 불가) |

생성 후 **Public IP 주소 메모** (예: `140.245.xxx.xxx`)

---

## STEP 2. 포트 방화벽 열기 (Security List)

Networking → Virtual Cloud Networks → goldsangsa → **Security 탭** → Default Security List for goldsangsa → **Add Ingress Rules**

| Protocol | Source CIDR | Port |
|----------|-------------|------|
| TCP | 0.0.0.0/0 | 80 |
| TCP | 0.0.0.0/0 | 443 |
| TCP | 0.0.0.0/0 | 3000 |

---

## STEP 3. SSH 접속

```bash
# .pem 파일 권한 설정 (Mac)
chmod 400 ~/Downloads/goldsangsa.pem

# 접속
ssh -i ~/Downloads/goldsangsa.pem ubuntu@140.245.xxx.xxx
```

---

## STEP 4. Docker 설치

```bash
sudo apt update && sudo apt upgrade -y

curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker ubuntu
newgrp docker

# 설치 확인
docker --version
docker compose version
```

---

## STEP 5. 프로젝트 파일 전송

로컬 터미널(Mac)에서 실행:

```bash
scp -i ~/Downloads/goldsangsa.pem -r /Users/choejeong-in/Gold ubuntu@140.245.xxx.xxx:~/goldsangsa
```

---

## STEP 6. docker-compose.yml 수정 (중요!)

서버에서 실행:

```bash
cd ~/goldsangsa
nano docker-compose.yml
```

아래 두 줄을 서버 IP로 변경:

```yaml
# frontend
NEXT_PUBLIC_API_URL: http://140.245.xxx.xxx:4000/api

# backend
FRONTEND_URL: http://140.245.xxx.xxx:3000
```

---

## STEP 7. 서버 내부 방화벽 열기

> Oracle Cloud는 서버 내부에도 iptables 방화벽이 별도로 존재 (필수)

```bash
sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 443 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 3000 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 4000 -j ACCEPT

# 재부팅 후에도 유지
sudo apt install -y iptables-persistent
sudo netfilter-persistent save
```

---

## STEP 8. 실행

```bash
cd ~/goldsangsa
docker compose up -d --build
```

브라우저에서 `http://140.245.xxx.xxx:3000` 접속 확인

---

## STEP 9. 도메인 연결

도메인 구매처(가비아 등) DNS 관리에서 A 레코드 추가:

| 타입 | 호스트 | 값 |
|------|--------|-----|
| A | @ | 140.245.xxx.xxx |
| A | www | 140.245.xxx.xxx |

전파까지 최대 24시간 소요 (보통 수십 분)

---

## STEP 10. Nginx + HTTPS 설정

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

Nginx 설정 파일 생성:

```bash
sudo nano /etc/nginx/sites-available/goldsangsa
```

아래 내용 붙여넣기:

```nginx
server {
    listen 80;
    server_name goldsangsa.com www.goldsangsa.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/goldsangsa /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# 무료 HTTPS 인증서 발급
sudo certbot --nginx -d goldsangsa.com -d www.goldsangsa.com
```

```
       → Next.js  :3000  (프론트엔드)
       → NestJS   :4000  (API)
       → PostgreSQL :5432 (DB, 내부 전용)
```
